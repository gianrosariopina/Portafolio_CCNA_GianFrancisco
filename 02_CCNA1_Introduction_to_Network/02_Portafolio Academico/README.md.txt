# Portafolio Académico – CCNA1: Introduction to Networks

Este repositorio forma parte del portafolio académico de **Gian Francisco**.  
Su objetivo es documentar y demostrar las competencias adquiridas durante el curso:

**CCNA 1 – Introduction to Networks**  
(Cisco Certified Network Associate – Networking Academy)

---

## 1. Contenido del repositorio

La información se encuentra organizada en carpetas para una navegación clara:

### **01_Presentaciones_Profesor – SOLO README (sin pptx)**
- Estructura y temas tratados durante el curso.
- **Nota:** Las presentaciones originales del profesor no se incluyen por confidencialidad.
- Se proporciona únicamente un archivo README con la descripción de los temas.

### **02_Preguntas_Resueltas**
- Documentos en formato `.zip` y `.md` que contienen:
  - Preguntas teóricas y prácticas respondidas.
  - Ejercicios realizados durante el curso.

### **03_Laboratorios_Wireshark**
- Análisis de tráfico de red capturado con Wireshark.
- Archivos en formato `.pcapng` y documentación explicativa.
- Objetivo: Comprender protocolos y comunicación en redes.

### **04_Labs_PacketTracer**
- Laboratorios desarrollados en **Cisco Packet Tracer**:
  - Archivos `.pkt`, `.pka`, `.pksz`.
  - Configuración de dispositivos y topologías de red.
- Todos los laboratorios están organizados y documentados en subcarpetas.

---

## 2. Archivos no incluidos en este repositorio

### **Archivos mayores a 100 MB**
- Debido a las limitaciones de GitHub, los archivos cuyo tamaño excede los 100 MB no se encuentran aquí.  
- Estos se almacenan en un espacio privado de Google Drive y podrán compartirse únicamente bajo solicitud formal.

### **Instalador de Cisco Packet Tracer**
- El instalador **NO está incluido** en este repositorio por razones de tamaño y licenciamiento.
- Para trabajar con los archivos `.pkt` o `.pka`, el interesado debe descargarlo directamente desde:
  > https://www.netacad.com/

---

## 3. Competencias demostradas en este repositorio

- **Fundamentos de redes:** conceptos básicos, direccionamiento IP, topologías.
- **Configuración de dispositivos:** routers, switches y PCs simuladas en Packet Tracer.
- **Resolución de problemas:** ejercicios prácticos y casos de laboratorio.
- **Herramientas de análisis:** uso de Wireshark para inspección de tráfico de red.
- **Documentación profesional:** organización estructurada, documentación clara en README y buenas prácticas de versionado.

---

## 4. Tecnologías y herramientas utilizadas

- **Cisco Packet Tracer** – Simulación de redes.
- **Wireshark** – Captura y análisis de tráfico.
- **GitHub** – Control de versiones y portafolio profesional.
- **Markdown (.md)** – Documentación técnica.
